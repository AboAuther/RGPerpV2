# 配置字典与默认值矩阵

## 1. 文档目标

本文档定义所有关键动态配置项的分组、作用域、默认值和变更要求。

## 2. 配置原则

1. 关键参数不得硬编码在业务逻辑；
2. 风控、提现、symbol 状态变更必须可审计；
3. 高风险配置必须支持审批与回滚；
4. 配置读取失败时系统应 fail closed。

## 3. 配置分组

## 3.1 Global

| key | 默认值 | 说明 |
| --- | --- | --- |
| `system.mode` | `review` | 当前环境模式 |
| `system.read_only` | `false` | 全站只读 |
| `system.reduce_only` | `false` | 全站 reduce-only |
| `system.trace_header_required` | `true` | 写请求要求 trace header |

## 3.2 Auth

| key | 默认值 | 说明 |
| --- | --- | --- |
| `auth.nonce_ttl_sec` | `300` | nonce 过期时间 |
| `auth.access_ttl_sec` | `3600` | access token 过期时间 |
| `auth.refresh_ttl_sec` | `2592000` | refresh token 过期时间 |
| `auth.max_failed_login_per_ip_per_hour` | `60` | 限流 |

## 3.3 Wallet

| key | 默认值 | 说明 |
| --- | --- | --- |
| `wallet.deposit.confirmations.ethereum` | `12` | ETH 确认数 |
| `wallet.deposit.confirmations.arbitrum` | `20` | Arbitrum 确认数 |
| `wallet.deposit.confirmations.base` | `20` | Base 确认数 |
| `wallet.withdraw.fee.usdc` | `1` | 默认提现手续费 |
| `wallet.withdraw.manual_review_threshold` | `10000` | 大额审核阈值 |
| `wallet.withdraw.daily_limit_per_user` | `50000` | 用户日累计提现限额 |
| `wallet.hot_wallet.min_balance` | `10000` | 热钱包最低余额 |
| `wallet.hot_wallet.max_balance` | `200000` | 热钱包最高余额 |

## 3.4 Trading

| --- | --- | --- |
| `trading.default_taker_fee_bps` | `5` | 吃单费率 |
| `trading.default_maker_fee_bps` | `2` | 挂单费率 |
| `trading.default_base_spread_bps` | `3` | 基础点差 |
| `trading.max_slippage_bps` | `100` | 默认最大滑点保护 |
| `trading.order_ttl_sec.gtd_default` | `86400` | 默认 GTD 过期 |

## 3.5 Risk

| key | 默认值 | 说明 |
| --- | --- | --- |

| `risk.mark_price_stale_sec` | `3` | 标记价格过期阈值 |
| `risk.force_reduce_only_on_stale_price` | `true` | 行情异常时降级 |
| `risk.liquidation_penalty_rate` | `0.01` | 强平罚金 |
| `risk.liquidation_extra_slippage_bps` | `20` | 强平成交额外滑点 |
| `risk.max_open_orders_per_user_per_symbol` | `20` | 每 symbol 最大挂单数 |

## 3.6 Funding

| key | 默认值 | 说明 |
| --- | --- | --- |
| `funding.interval_sec` | `3600` | 资金费率结算周期 |
| `funding.cap_rate_per_hour` | `0.0075` | 小时资金费率上限 |
| `funding.min_valid_source_count` | `1` | 最小有效源数 |
| `funding.default_model.crypto` | `EXTERNAL_AVG` | Crypto 默认模型 |

## 3.7 Hedge

| key | 默认值 | 说明 |
| --- | --- | --- |
| `hedge.enabled` | `true` | 是否启用对冲 |
| `hedge.soft_threshold_ratio` | `0.2` | 软阈值 |
| `hedge.hard_threshold_ratio` | `0.4` | 硬阈值 |
| `hedge.max_retry` | `5` | 对冲最大重试 |
| `hedge.degrade_to_reduce_only_on_failure` | `true` | 连续失败降级 |

## 3.8 Review

| key | 默认值 | 说明 |
| --- | --- | --- |
| `review.faucet.enabled` | `true` | faucet 开关 |
| `review.faucet.amount_per_request` | `10000` | 每次 faucet 额度 |
| `review.faucet.max_requests_per_day` | `10` | 日次数限制 |
| `review.mock_market_data.enabled` | `true` | mock 行情 |

## 4. Scope

支持 scope：

- `global`
- `chain`
- `symbol`
- `asset_class`
- `user_group`

优先级：

```text
symbol > asset_class > chain > global
```

## 5. 生效策略

### 5.1 即时生效

- UI 开关
- 非风控展示配置

### 5.2 下个安全点生效

- funding interval
- symbol 状态
- 风险阈值

### 5.3 必须审批

- 提现限额
- risk 参数
- hedge 阈值
- symbol 停开/停充/停提

## 6. 读取与缓存

- API 和 worker 启动时加载配置快照
- 热配置从 MySQL 读取并放入 Redis 缓存
- 缓存失效时回源 MySQL
- MySQL 不可用时关键风控配置应走本地只读快照并 fail closed

## 7. 审计要求

每项配置变更必须记录：

- `config_key`
- `scope`
- `old_value`
- `new_value`
- `reason`
- `created_by`
- `approved_by`
- `effective_at`
- `rollback_of`

## 8. 禁止事项

- 禁止业务代码直接硬编码风控和费率参数；
- 禁止无审计变更；
- 禁止在生产环境开启 faucet；
- 禁止配置加载失败后默认放宽风控。
