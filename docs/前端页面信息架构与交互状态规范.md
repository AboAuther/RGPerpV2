# 前端页面信息架构与交互状态规范

## 1. 文档目标

本文档定义前端模块拆分、页面信息架构、接口映射、交互状态与安全边界。

前端原则：

1. 前端不持有资金真相；
2. 前端不做最终权限判断；
3. 前端必须清晰展示处理中、失败、已确认、待确认状态；
4. 前端不得用乐观更新掩盖风险链路不确定性。

## 4. 页面与接口映射

### 4.1 登录页

调用：

- `POST /api/v1/auth/nonce`
- `POST /api/v1/auth/login`

状态：

- 请求 nonce 中
- 等待钱包签名
- 验签中
- 登录成功
- 登录失败

### 4.2 交易页

调用：

- `GET /api/v1/markets/symbols`
- `GET /api/v1/markets/tickers`
- `POST /api/v1/orders`
- `POST /api/v1/orders/:orderId/cancel`
- `GET /api/v1/orders`
- `GET /api/v1/positions`
- 基于 HTTP 的定时轮询与写后重载

必须展示：

- 可交易余额
- 权益
- 占用保证金
- 未实现 PnL
- symbol 状态
- 是否 `reduce-only`
- 行情是否延迟

### 4.3 充值页

调用：

- `GET /api/v1/wallet/deposit-addresses`
- `GET /api/v1/wallet/deposits`

必须展示：

- 每链独立充值地址
- 支持资产
- 确认数说明
- `DETECTED / CONFIRMING / CREDITED` 状态

### 4.4 提现页

调用：

- `POST /api/v1/wallet/withdrawals`
- `GET /api/v1/wallet/withdrawals`

必须展示：

- 可提现余额
- 提现手续费
- 审核状态
- 链上状态
- 失败退款状态

### 4.5 Portfolio 页

调用：

- `GET /api/v1/account/summary`
- `GET /api/v1/account/balances`
- `GET /api/v1/account/risk`
- `GET /api/v1/positions`

### 4.6 Explorer

调用：

- `GET /api/v1/explorer/events`
- `GET /api/v1/explorer/events/:eventId`
- `GET /api/v1/explorer/ledger/:ledgerTxId`
- `GET /api/v1/explorer/tx/:chainTxHash`

## 5. 前端状态模型

## 5.1 只读状态来源

以下状态仅以后端返回为准：

- balance
- equity
- available balance
- order status
- position status
- funding history
- liquidation status
- deposit/withdraw status

## 5.2 本地暂存状态

允许前端本地保存：

- 表单输入
- 当前选中的 symbol
- UI filter/sort
- 短期请求 loading 状态

## 5.3 禁止的本地真相

- 禁止本地直接推导“提现已完成”
- 禁止本地直接把下单请求视为成交成功
- 禁止本地持久化 access token 于不安全存储

## 6. 交互状态规范

### 6.1 下单

按钮态：

- 可点击
- 校验中
- 提交中
- 已受理
- 已拒绝

展示要求：

- `ACCEPTED` 不等于 `FILLED`
- `TRIGGER_WAIT` 必须明确展示“等待触发”
- `SYSTEM_CANCELED` 必须明确说明由风控/清算触发

### 6.2 提现

状态标签：

- 已申请
- 已冻结
- 风控审核中
- 待签名/待广播
- 链上确认中
- 完成
- 失败已退回

### 6.3 充值

状态标签：

- 已检测
- 确认中
- 已到账
- 已归集
- 重组回滚

## 7. 刷新规则

### 7.1 当前阶段策略

- 交易页通过定时轮询刷新 `tickers`、订单、仓位、账户概览
- 钱包、Portfolio、History、Explorer 页面通过进入页面、手动刷新和页面恢复前台时重载

### 7.2 前端处理原则

- 以服务端返回为唯一真相
- 写操作成功后必须重载相关只读数据
- 若请求失败、断网恢复或页面回到前台，必须主动刷新关键页面数据
- WS 推送仅做刷新，不应成为唯一真相

## 8. 安全要求

### 8.1 登录

- nonce 不在前端复用
- 签名消息必须展示域名与用途
- 切换链后必须重新校验链环境

### 8.2 权限

- 前端路由守卫只做 UX，不做最终授权
- 管理页面每个操作必须由后端再次校验 RBAC

### 8.3 表单

- 金额、价格、数量必须前端校验格式
- 但不依赖前端校验作为最终约束

## 9. 失败与降级体验

### 9.1 行情异常

- 交易页显著显示“行情延迟/异常”
- 禁止用户误以为还能正常开仓

### 9.2 风控降级

- 若 symbol 进入 `REDUCE_ONLY`，下单面板必须锁定开仓选项
- 若账户清算中，必须明确显示并锁定风险增加动作

### 9.3 服务异常

- 用户操作失败必须显示 `trace_id` 或请求编号
- 不得仅提示“系统错误”而无可追踪信息
