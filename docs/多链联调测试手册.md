# RGPerp 本地多链充值联调手册

## 1. 目标

本文档用于在本地同时拉起三条 Anvil 链，并验证 Ethereum / Arbitrum / Base 三条链的 USDC 充值地址生成、链上转入、Indexer 检测和账本入账链路。

## 2. 本地链规划

- Local Ethereum: `http://127.0.0.1:8545` / `chain_id=31337`
- Local Arbitrum: `http://127.0.0.1:8546` / `chain_id=31338`
- Local Base: `http://127.0.0.1:8547` / `chain_id=31339`

三条链共用同一套 Anvil mnemonic，因此管理员与测试用户地址在三条链保持一致。

## 3. 启动步骤

在项目根目录执行：

```bash
bash deploy/scripts/start-local-multichain.sh
bash deploy/scripts/bootstrap-local-multichain.sh
docker compose up -d mysql mysql-init redis rabbitmq
bash deploy/scripts/run-backend-local.sh api-server
bash deploy/scripts/run-backend-local.sh indexer
```

如果需要彻底清理本地状态：

```bash
bash deploy/scripts/reset-local-dev.sh
```

## 4. 多链充值 smoke test

在 `api-server` 和 `indexer` 启动后执行：

```bash
bash deploy/scripts/smoke-test-multichain-local.sh
```

预期结果：

- 同一用户在三条链都能成功生成充值地址
- 三条链的 `routerOfUser(user_id)` 与 API 返回地址一致
- 三条链都能产生 `CREDITED` 充值记录
- 用户 `USER_WALLET/USDC` 余额累计增加 `6006`

## 5. 生成物

- 多链链上配置 env: `deploy/env/local-chains.env`
- 多链 deployment manifest: `.local/manifest/dev/*.json`
- 前端本地 env: `frontend/.env.local`
