# RGPerp 本地页面联调测试手册

## 1. 适用范围

本文档用于本地开发环境下的页面联调测试，覆盖以下已打通链路：

- 浏览器钱包登录
- 账户总览查询
- Local Anvil 链充值地址生成与本地测试工具
- 本地测试 ETH / USDC 领取
- 充值监听、确认入账
- 常规提现自动广播与链上完成
- 高风险提现人工复核后完成
- Explorer 充值 / 提现事件查询

当前文档不覆盖以下未完成或未在页面接通的能力：

- 交易、撮合、仓位、风控、清算、资金费、对冲
- 完整管理后台页面审批链路

## 3. 启动前检查

### 3.1 启动本地链与后端

在项目根目录先启动本地链与合约：

```bash
sh deploy/scripts/start-local-chain.sh
sh deploy/scripts/bootstrap-local-chain.sh
```

再启动数据层和后端进程：

```bash
docker compose up -d mysql mysql-init redis rabbitmq api-server indexer
```

前端单独启动：

```bash
cd frontend
pnpm dev
```

### 3.2 服务健康检查

后端健康检查：

```bash
curl http://127.0.0.1:8080/healthz
```

预期返回：

```json
{"code":"OK","message":"ok","trace_id":"trace_generated","data":{"status":"OK"}}
```

前端首页检查：

```bash
curl http://127.0.0.1:5173
```

预期返回 HTML。

## 6. 页面联调步骤

## 6.1 登录

访问：

- `http://127.0.0.1:5173/login`

操作步骤：

1. 点击“连接钱包并登录”。
2. 在钱包中切换到 `RGPerp Local` 网络。
3. 选择普通用户地址 `0x70997970C51812dc3A010C7d01b50e0d17dc79C8`。
4. 完成一次 `personal_sign` 签名。

预期结果：

- 页面跳转到 `/portfolio`
- 顶部导航可进入 Wallet、Explorer 等页面
- 登录后不应报 `UNAUTHORIZED`

## 6.2 账户总览

访问：

- `http://127.0.0.1:5173/portfolio`

预期结果：

- 可看到账户 summary
- 初次登录时余额可能为 `0`
- 若已执行过本地 mint 或充值，`equity` 与 `available_balance` 应大于 `0`

## 6.3 充值页

访问：

- `http://127.0.0.1:5173/wallet/deposit`

重点校验：

- 能加载充值地址列表
- 能展示确认数要求
- 能展示最近充值记录
- 仅 `CREDITED` 代表已到账

### 推荐测试方式：本地工具 + 便捷充值
在 `Local Anvil / USDC` 卡片上依次点击：

- `生成充值地址`
- `显示地址`
- `领取测试 ETH`
- `领取测试 USDC`
- `便捷充值 1000 USDC`

预期结果：

- 页面提示链上操作已提交
- 充值记录出现一条新的 `deposit`
- 状态最终应为 `CREDITED`
- 随后在账户页可看到余额增加

说明：

- 这是当前页面最稳定的充值联调路径。
- `领取测试 USDC` 通过前端钱包直连本地链 USDC 合约。
- `便捷充值 1000 USDC` 会执行真实 `transfer + forward`，随后由 indexer 入账。

## 6.4 提现页

访问：

- `http://127.0.0.1:5173/wallet/withdraw`

测试输入建议：

- 链: `Local Anvil (31337)`
- 资产: `USDC`
- 数量: `10`
- 目标地址: `0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266`

预期结果：

- 提交成功后提示“资金已冻结，常规提现会自动广播，特殊风控才人工复核”
- 提现历史新增一条记录
- 常规小额提现会自动从 `APPROVED` 推进到 `COMPLETED`
- 命中人工复核阈值时会先进入 `RISK_REVIEW`

注意：

- `USER_WITHDRAW_HOLD` 表示资金已经冻结，但尚未离开平台账本。
- 只有到 `BROADCASTED / COMPLETED` 才表示链上提现已真正推进。
- 高风险提现仍需管理员调用审批 API 完成放行。

## 6.5 Explorer

访问：

- `http://127.0.0.1:5173/explorer`

测试方式：

1. 先完成一次充值或提现。
2. 打开 Explorer 页面并点击“刷新事件”。
3. 用以下关键字搜索：
   - 充值或提现的 `tx_hash`
   - `WITHDRAW_HOLD`
   - `WITHDRAW_BROADCAST`
   - `WITHDRAW_COMPLETE`

预期结果：

- 可以看到 `ledger.committed` 类事件
- 充值事件包含 `Time / Asset / Amount / Chain Tx`
- 提现事件可按真实 `tx_hash` 搜到，且能看到 `WITHDRAW_HOLD / WITHDRAW_BROADCAST / WITHDRAW_COMPLETE`

## 7. 推荐完整测试链路

建议按以下顺序进行：

1. 登录普通用户钱包
2. 打开 `/wallet/deposit`
3. 在 `Local Anvil / USDC` 上依次点击 `领取测试 USDC` 和 `便捷充值 1000 USDC`
4. 等待充值记录状态为 `CREDITED`
5. 打开 `/portfolio`，确认余额增加
6. 打开 `/wallet/withdraw`
7. 提交一笔 `10 USDC` 提现到管理员地址，等待状态自动到 `COMPLETED`
8. 复制该笔提现的 `tx_hash`
9. 打开 `/explorer`，用 `tx_hash` 搜索，确认出现新的提现事件

## 8. 当前已知限制

### 8.1 管理后台页面暂不作为页面联调范围

虽然管理员审批 API 已存在，但当前前端管理员路由依赖登录响应中的管理员标识：

- `user.is_admin`
- `user.role`
- `user.capabilities`

当前登录响应尚未把这些字段完整返回到前端，因此：

- `/admin/dashboard`
- `/admin/withdrawals`
- `/admin/configs`
- `/admin/liquidations`

不作为本轮页面联调验收范围。

### 8.2 提现复核页仍以 API 为主

当前已打通：

- 常规提现自动广播与完成
- 高风险提现进入 `RISK_REVIEW`
- 后端管理员审批 API 放行后继续自动完成

当前未打通：

- 完整后台审批页面

### 8.3 交易页仍为壳层

`/trade` 当前不作为真实交易联调页面，不纳入本手册验收范围。

## 9. 失败排查

### 9.1 登录失败

检查项：

- 钱包是否切换到 `Chain ID 31337`
- 是否使用了本文档提供的本地测试账户
- 后端是否可访问 `http://127.0.0.1:8080/healthz`

### 9.2 本地 mint 失败

检查项：

- 当前登录地址是否与请求地址一致
- 当前环境是否为 `review`
- 页面是否连接真实后端而非错误 provider

### 9.3 页面无法看到充值或余额未变化

检查项：

- 是否使用 `Local Mock(31337)` 路径
- 是否等待到充值状态变为 `CREDITED`
- 是否刷新 `/portfolio` 与 `/wallet/deposit`

### 9.4 提现失败

检查项：

- 提现金额是否超过 `USER_WALLET` 可用余额
- 地址是否是合法 EVM 地址
- 浏览器请求头是否正常携带会话令牌

## 10. 当前页面联调验收标准

满足以下条件即可认为“页面基础联调通过”：

- 普通用户可通过浏览器钱包完成登录
- 账户总览可正常展示
- Deposit 页面可正常加载 Local Mock 充值地址
- 本地 mint 可新增一笔 `CREDITED` 充值
- 提现页面可成功提交提现申请
- 常规提现应自动完成
- 提现 `tx_hash` 应可在 Explorer 搜到
- Explorer 可看到对应的账本事件
