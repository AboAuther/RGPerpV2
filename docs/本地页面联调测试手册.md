# RGPerp 本地页面联调测试手册

## 1. 适用范围

本文档用于本地开发环境下的页面联调测试，覆盖以下已打通链路：

- 浏览器钱包登录
- 账户总览查询
- Local Mock 链充值地址展示
- 本地测试 USDC mint
- 提现申请
- Explorer 事件查询

当前文档不覆盖以下未完成或未在页面接通的能力：

- 提现广播上链与链上完成
- 交易、撮合、仓位、风控、清算、资金费、对冲
- 管理后台页面审批链路

## 6. 页面联调步骤

## 6.1 登录

访问：

- `http://127.0.0.1:5173/login`

操作步骤：

1. 点击“连接钱包”。
2. 在钱包中切换到 `RGPerp Local Mock` 网络。
3. 选择普通用户地址 `0x70997970C51812dc3A010C7d01b50e0d17dc79C8`。
4. 点击“请求 Nonce”。
5. 点击“登录”并完成 `personal_sign` 签名。

预期结果：

- 页面跳转到 `/portfolio`
- 顶部导航可进入 Wallet、Explorer 等页面
- 登录后不应报 `UNAUTHORIZED`

## 6.2 账户总览

访问：

- `http://127.0.0.1:5173/portfolio`

预期结果：

- 可看到账户 summary
- 初次登录时余额可能为 `0`
- 若已执行过本地 mint 或充值，`equity` 与 `available_balance` 应大于 `0`

## 6.3 充值页

访问：

- `http://127.0.0.1:5173/wallet/deposit`

重点校验：

- 能加载充值地址列表
- 能展示确认数要求
- 能展示最近充值记录
- 仅 `CREDITED` 代表已到账

### 推荐测试方式：本地 mint 1000 USDC
在 `Local Mock / USDC` 卡片上点击：

- `mint 1000 USDC`

预期结果：

- 页面提示“已向该地址 mint 1000 USDC”
- 充值记录出现一条新的 `deposit`
- 状态最终应为 `CREDITED`
- 随后在账户页可看到余额增加

说明：

- 这是当前页面最稳定的充值联调路径。
- 按钮背后会在本地链上向该地址 mint `1000 USDC`；若该地址是 router，后端会继续触发 `forward` 进入充值链路。

## 6.4 提现页

访问：

- `http://127.0.0.1:5173/wallet/withdraw`

测试输入建议：

- 链: `Local Mock (31337)`
- 资产: `USDC`
- 数量: `10`
- 目标地址: `0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266`

预期结果：

- 提交成功后提示“提现已申请，请等待审核与链上状态推进”
- 提现历史新增一条记录
- 当前已实现状态通常为 `APPROVED` 之前的页面可见链路，后端真实审批后可查询到 `APPROVED`

注意：

- 当前页面只覆盖提现申请。
- 提现广播、链上确认、完成回填尚未接入页面闭环。

## 6.5 Explorer

访问：

- `http://127.0.0.1:5173/explorer`

测试方式：

1. 先完成一次本地 mint 或提现申请。
2. 打开 Explorer 页面并点击“刷新事件”。
3. 用以下关键字搜索：
   - `WITHDRAW_HOLD`
   - `DEPOSIT`
   - `ledger_tx_id`
   - `deposit_id`
   - `withdraw_id`

预期结果：

- 可以看到 `ledger.committed` 类事件
- 新充值会出现 `DEPOSIT_DETECTED`、`DEPOSIT`
- 新提现会出现 `WITHDRAW_HOLD`

## 7. 推荐完整测试链路

建议按以下顺序进行：

1. 登录普通用户钱包
2. 打开 `/wallet/deposit`
3. 在 `Local Mock / USDC` 上点击 `mint 1000 USDC`
4. 等待充值记录状态为 `CREDITED`
5. 打开 `/portfolio`，确认余额增加
6. 打开 `/wallet/withdraw`
7. 提交一笔 `10 USDC` 提现到管理员地址
8. 打开 `/explorer`，确认出现新的提现账本事件

## 8. 当前已知限制

### 8.1 管理后台页面暂不作为页面联调范围

虽然管理员审批 API 已存在，但当前前端管理员路由依赖登录响应中的管理员标识：

- `user.is_admin`
- `user.role`
- `user.capabilities`

当前登录响应尚未把这些字段完整返回到前端，因此：

- `/admin/dashboard`
- `/admin/withdrawals`
- `/admin/configs`
- `/admin/liquidations`

不作为本轮页面联调验收范围。

### 8.2 真实链上提现完成未接通

当前已打通：

- 提现申请
- 后端审批 API
- 账本冻结与审计事件

当前未打通：

- 广播上链
- 链上确认
- 完成回填

因此页面上不应把提现视为“最终完成”。

### 8.3 交易页仍为壳层

`/trade` 当前不作为真实交易联调页面，不纳入本手册验收范围。

## 9. 失败排查

### 9.1 登录失败

检查项：

- 钱包是否切换到 `Chain ID 31337`
- 是否使用了本文档提供的本地测试账户
- 后端是否可访问 `http://127.0.0.1:8080/healthz`

### 9.2 本地 mint 失败

检查项：

- 当前登录地址是否与请求地址一致
- 当前环境是否为 `review`
- 页面是否连接真实后端而非错误 provider

### 9.3 页面无法看到充值或余额未变化

检查项：

- 是否使用 `Local Mock(31337)` 路径
- 是否等待到充值状态变为 `CREDITED`
- 是否刷新 `/portfolio` 与 `/wallet/deposit`

### 9.4 提现失败

检查项：

- 提现金额是否超过 `USER_WALLET` 可用余额
- 地址是否是合法 EVM 地址
- 浏览器请求头是否正常携带会话令牌

## 10. 当前页面联调验收标准

满足以下条件即可认为“页面基础联调通过”：

- 普通用户可通过浏览器钱包完成登录
- 账户总览可正常展示
- Deposit 页面可正常加载 Local Mock 充值地址
- 本地 mint 可新增一笔 `CREDITED` 充值
- 提现页面可成功提交提现申请
- Explorer 可看到对应的账本事件
